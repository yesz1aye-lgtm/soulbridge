# fix_request

사용자 모드에서 처리하지 못한 문제·개선 요청을 기록하는 폴더.

## 형식

파일명: `YYYY-MM-DD_간단한제목.md`

내용 템플릿:

```markdown
# 발생일: 2026-04-27
# 모드: 사용자
# 상태: pending  (pending | in_progress | done)

## 무슨 일이 있었나
(사용자가 무엇을 시켰고, Claude가 무엇을 시도했는지)

## 원본 입력
(파일 경로, 사용자가 준 자료 요약)

## 기대 결과
(사용자가 원했던 것)

## 실제 결과
(에러 메시지, 잘못된 출력, 또는 막힌 지점)

## 사용자가 추가로 알려준 정보
(있다면)

## 추정 원인 (선택)
(Claude가 짐작한 것이 있다면)
```

## 처리 후

개발자가 처리한 항목은 `done/` 하위로 이동하고 파일 상단에 처리 요약·해결 커밋/변경사항을 추가한다.
